## Project

**Tertiary Scanner** — a native iOS document scanner (Swift 6 / SwiftUI / iOS 18+, MVVM, fully
offline). Scan with VisionKit, enhance with Core Image filters, OCR with Vision, export PDF/JPG
to Photos, Files, or iCloud Drive. No backend; everything runs on-device.

- Bundle ID: `com.tertiaryinfotech.scannerapp` · App Store name: **Tertiary Scanner**
- Team: `GU9WTSTX9M` (Tertiary Infotech) · App Store Connect App ID: `6779988762`

## Build & run

The Xcode project is **generated by XcodeGen** from `project.yml` — do not hand-edit
`DocumentScannerApp.xcodeproj` (it is gitignored and regenerated).

```bash
xcodegen generate                         # regenerate the project after adding/removing files
# Simulator build:
xcodebuild -project DocumentScannerApp.xcodeproj -scheme DocumentScannerApp \
  -destination 'platform=iOS Simulator,name=iPhone 17' build CODE_SIGNING_ALLOWED=NO
# Device / release archive uses automatic signing with DEVELOPMENT_TEAM=GU9WTSTX9M.
```

**Important:** XcodeGen uses an explicit file list. After creating a new `.swift` file, run
`xcodegen generate` or the build won't see it (symptom: "cannot find X in scope").

## Architecture (MVVM)

```
App/          DocumentScannerApp.swift  — @main, SwiftData ModelContainer
Models/       ScanDocument, ScanPage (@Model), FilterType
Services/     ScannerService (VisionKit), OCRService (Vision), PDFService,
              ExportService (Photos/Files), StorageService (files + SwiftData)
ViewModels/   ScannerViewModel (capture→edit→save), LibraryViewModel   (@MainActor @Observable)
Views/        Home, Scanner, ScanEditor, Preview, Filter, Export, Library,
              DocumentDetail, Settings + Components/ (ShareSheet, DocumentExporter, …)
Utilities/    ImageProcessor (Core Image), SettingsStore, Constants, DemoSeed (DEBUG)
Resources/    Assets.xcassets (AppIcon, AccentColor), PrivacyInfo.xcprivacy
```

### Conventions & gotchas
- **Persistence**: document/page *metadata* in SwiftData; page *images, thumbnails, PDFs* are
  files under Application Support (`Constants.scansDirectory`). Models store filenames only.
- **Swift 6 strict concurrency** is on. ViewModels are `@MainActor @Observable`. Heavy work
  (Core Image, OCR, PDF) runs in async services; cross actor boundaries with `Data`/`CGImage`
  (UIImage is non-Sendable). `ImageProcessor` is a `@unchecked Sendable` shared `CIContext`.
- **VisionKit delegate** (`VNDocumentCameraViewControllerDelegate`) is NOT `@MainActor` in the
  SDK — its Coordinator must stay nonisolated. The PHPicker Coordinator may be `@MainActor`.
- **Simulator has no camera** → the Scan button falls back to `PhotoImportView` (PHPicker).
  `DemoSeed` (DEBUG, env `SCANNER_SEED_DEMO=1`) populates sample docs for screenshots/testing.
- New filters: add a `FilterType` case + a Core Image graph in `ImageProcessor.output(for:)`.

## App Store / release

- `.env` (gitignored) holds App Store Connect API creds; the `.p8` lives outside the repo.
  Load with `set -a; source .env; set +a`.
- Submission workflow + scripts: `.claude/skills/app-store-submission/` (asc_submit.py).
  App creation, App Privacy, and age rating are web-UI-only (API forbids them).
- Pre-flight for release: bump `CURRENT_PROJECT_VERSION` in `project.yml`, flat 1024 icon
  (no alpha), `ITSAppUsesNonExemptEncryption=false`, `PrivacyInfo.xcprivacy` present.
